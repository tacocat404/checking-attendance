# 출결기록실 · Attendance Archive

> 규칙 기반(VBA) vs LLM 증강(Make + Gemini) 출결 관리 시스템 — 프론트엔드
> 실무프로젝트 1 (실프랩 1) · 2026

Google Forms → Sheets → Gemini → Gmail 로 이어지는 **Make.com 노코드 자동화 파이프라인**의
**웹 프론트엔드**입니다. 직원이 버튼 한 번으로 출퇴근을 기록하면 Google Forms에 제출되고,
이후 안내 문구 생성(Gemini)과 메일 발송(Gmail)·로그 기록까지 자동으로 처리됩니다.
축적된 Google Sheets 데이터를 화면에서 통계·명단·메일 이력으로 보여줍니다.

## 화면 구성

| # | 페이지 | 설명 |
|---|--------|------|
| 01 | **오늘** (`index.html`) | 실시간 시계 + 출근/퇴근 체크 버튼, 오늘 출결 요약, 확인이 필요한 인원(지각·결근) 원장 |
| 02 | **근태 내역** (`history.html`) | 전체 출결 기록 원장 — 상태·날짜·이름 필터 |
| 03 | **통계** (`stats.html`) | 일별 출결 현황, 개인별 누적 기록 |
| 04 | **메일 이력** (`mail.html`) | Gemini가 작성하고 Gmail이 보낸 안내 메일 발송 대장 |

## 자동화 파이프라인

백엔드는 **Make.com 노코드 시나리오**로, 코드 없이 모듈 5개를 이어 만들었습니다.
새 출결 응답이 들어오면 시나리오 전체가 자동으로 실행됩니다.

```
[출결기록실 웹]  →  Forms  →  Sheets  →  Gemini  →  Gmail  →  Sheets(로그)
   출퇴근 버튼      응답 감지   응답 기록   문구 생성   안내 발송    발송 로그
       ↑                                                          │
       └──────────  화면에 통계·명단·메일 이력으로 표시  ←──────────┘
```

| 순서 | 모듈 | 동작 |
|------|------|------|
| 1 | Google Forms — Watch Responses | 출퇴근 응답 감지 (트리거) |
| 2 | Google Sheets — Add a Row | 응답을 출결 시트에 기록 |
| 3 | Google Gemini — Generate a response | 개인화 안내 문구 생성 |
| 4 | Gmail — Send an Email | 생성된 문구로 안내 메일 발송 |
| 5 | Google Sheets — Add a Row | 발송 내용을 로그 시트에 누적 |

> 계획서에는 규칙 기반 판정 라우터(3단계)와 지각·결근자 분류 시트, 그리고
> **VBA vs Make 판정 일치율 비교 실험**이 있었습니다. 이번 구현 범위에서는 위 5개 모듈까지를
> 완성했고, 판정 일치율 비교는 VBA 판정 데이터가 축적된 뒤의 과제로 남겨 두었습니다.

## 규칙 기반 vs LLM 증강 — 무엇이 달라졌나

| 구분 | 기존 · 엑셀 VBA 매크로 | 신규 · 출결기록실 (Make + Gemini) |
|------|------------------------|-----------------------------------|
| 처리 | 파일을 열고 매크로를 실행해야 처리 | 응답이 들어오는 즉시 시나리오 자동 실행 |
| 안내 | 개인화 피드백 없음, 안내는 수작업 | Gemini가 상황별 문구 작성, Gmail이 자동 발송 |
| 기록 | 결과가 엑셀 파일 내부에 고립 | Sheets에 응답·발송 로그 누적, 지표화 가능 |
| 확인 | 파일을 가진 사람만 열람 | 웹 4개 화면(통계·명단·메일 이력)에서 열람 |

## 디자인

에디토리얼(신문 지면) 컨셉 · "인주 스탬프" 색감 — 웜 페이퍼 배경, 차콜 잉크, 도장 인주색 포인트.
직각 레이아웃, 괘선 구획, 스크롤에 따라 천천히 떠오르는 리빌 애니메이션.

- 폰트: [Pretendard](https://github.com/orioncactus/pretendard)
- 차트/라이브러리 의존성 없음 (순수 HTML/CSS/JS)

## 설정 — `js/config.js`

연동 정보는 **`js/config.js` 한 파일에만** 있습니다. 값을 비워두면 데모 데이터로 작동합니다.

- **FORM** — Google Forms ID + 질문별 entry 번호 (이름·이메일·출퇴근 구분·시간)
- **SHEET** — Google Sheets ID + 탭 gid (폼 응답 / 발송·판정 로그 / VBA 판정 — 마지막 탭은 향후 비교 기능용으로 예약)
- **WORK** — 출근 마감 시각 (화면 표시용 지각 근사 판정 기준)

## 실행

정적 사이트입니다. 로컬에서 보려면 폴더에서:

```bash
python -m http.server 4173
# 브라우저에서 http://localhost:4173 접속
```

> ⚠️ **공개 저장소 주의** — `js/config.js`의 Google Forms/Sheets ID가 코드에 포함되어 있습니다.
> 연동된 시트는 "링크가 있는 사용자 열람" 상태여야 통계가 표시됩니다. 개인정보 보호를 위해,
> 공개 주소(github.io 등)에서는 자동으로 **데모 데이터만** 표시하고 실명·이메일이 담긴 실데이터는
> 본인 PC(localhost)에서만 읽습니다. (`js/data.js`의 `isLocalEnvironment()`)
