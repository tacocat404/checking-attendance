# 운영 노트

배포·데이터와 관련해 팀원이 알아둬야 할 내용입니다.

## 데이터 표시 방식 — 위치에 따라 자동으로 갈립니다

이 사이트는 **접속한 위치**에 따라 보여주는 데이터가 달라집니다. 설정을 따로 바꿀 필요 없이 코드가 알아서 판별합니다. (`js/data.js`의 `isLocalEnvironment()`)

| 접속 위치 | 표시 데이터 | 용도 |
|-----------|-------------|------|
| 본인 PC (`localhost` / `127.0.0.1` / 파일 직접 열기) | **실제 Google Sheets 데이터** | 작업·확인 |
| 인터넷 공개 주소 (`github.io` 등) | **데모(예시) 데이터** | 발표·공유 |

- 더미 데이터는 `index.html`에 박혀 있는 게 아니라, "실제 시트를 읽지 않을 때 대신 보여주는 예시"입니다.
- 공개 사이트에서 데모로 표시하는 이유: **반 친구들의 실명·이메일·출결 기록이 아무에게나 노출되지 않도록** 하기 위함입니다.
- 로컬에서 실제 데이터를 보려면 폴더에서 `python -m http.server 4173` 실행 후 `http://localhost:4173` 로 접속하세요.

## 주의 1 — 출퇴근 버튼은 공개 사이트에서도 실제로 제출됩니다

화면에 보이는 **숫자·명단만** 데모로 가린 것이고, **출근/퇴근 버튼 자체는 위치와 상관없이 실제 Google Forms로 전송**됩니다. (연결된 폼이 원래 "누구나 응답 가능" 상태이기 때문)

- 장점: 발표 때 버튼을 누르면 실제로 시트에 기록되는 걸 보여줄 수 있음
- 단점: 공개 주소에 아무나 들어와 가짜 출결을 넣을 수 있음
- 막으려면: 공개 사이트에서는 버튼을 비활성화하거나 "데모 안내"만 뜨게 처리 가능 (**결정 대기**)

## 주의 2 — 폼/시트 ID가 코드에 포함되어 있습니다

`js/config.js`에 Google Forms ID와 Sheets ID가 들어 있고, 저장소가 공개라 이 값들은 노출됩니다.

- 마음먹으면 그 ID로 시트를 직접 열어볼 수 있음 (시트가 "링크 열람" 공개 상태라서)
- 완전히 막으려면: 시트 공유를 "특정 사용자만"으로 변경 → 단, 그러면 로컬 실데이터 연동도 별도 인증이 필요해짐 (**결정 대기**)

## 백엔드팀 전달 사항

- **Make "Add a Row"가 빈 행을 남김**: 폼 응답 탭(gid 88862461)에 타임스탬프만 있고 이름 등이 비어 있는 행이 쌓임. 화면에서는 걸러내지만 시트가 지저분해짐 — 모듈 설정 확인 필요.
- **VBA 판정 탭이 비어 있음**(gid 1061791224): 이 데이터가 채워지면 기획서의 핵심 지표인 **"VBA vs Make 판정 일치율" 비교 화면**을 만들 수 있음. 판정·발송 로그 탭(gid 276345751)의 VBA판정·일치여부·Gemini문구 열도 함께 채워지면 메일 이력에 실제 Gemini 문구가 표시됨.
